# Contributing to Proof of Delegation

Thanks for your interest in contributing! PoD is an early-stage open standard and
every perspective matters — whether you're a security researcher, web developer,
agent builder, or e-commerce operator.

## Ways to Contribute

### Spec Feedback
- Open an issue with the `spec` label
- Discuss design decisions and trade-offs
- Propose new scopes or verticals
- Challenge our assumptions

### Code
- Improve the Python reference implementation
- Build implementations in other languages (Node.js, Go, Rust)
- Add tests and improve coverage
- Build integrations with web frameworks (FastAPI, Express, etc.)

### Documentation
- Improve README and guides
- Write tutorials
- Create diagrams and visual explanations
- Translate to other languages

### Real-World Testing
- Try implementing PoD in your project
- Report friction points and usability issues
- Share use cases we haven't considered

## Development Setup

```bash
git clone https://github.com/deegerhq/wais-pod.git
cd wais
pip install -e ".[dev]"
pytest
```

## Pull Request Process

1. Fork the repo and create your branch from `main`
2. Add tests for any new functionality
3. Ensure all tests pass
4. Update documentation if needed
5. Open a PR with a clear description

## Code of Conduct

Be respectful, be constructive, be open-minded. We're building something new
and there are no bad questions.
